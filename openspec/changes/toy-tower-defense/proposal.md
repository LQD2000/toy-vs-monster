## Why

开发一款玩具题材的塔防游戏，融合《植物大战僵尸》的核心塔防玩法与《美食大战老鼠》的升级升阶系统，创造一款有长期养成深度、支持双人合作的 2D 跑道式塔防游戏。选择玩具题材是因为其天然适配"升级改装"的叙事，且品类丰富、风格轻松，适合广泛的玩家群体。

## What Changes

**核心玩法：**
- 2D 跑道式塔防，5-6 条固定路径，进攻方从右侧出发，向左侧前进
- 每条跑道左侧尽头有一辆小车（一次性防御），首次有怪物到达时触发，清除整行怪物后消失
- 防守方为各类玩具，进攻方为怪兽（普通敌人）+ 黑暗玩具（精英/Boss）
- 局内资源为"电力"，通过放置产电玩具获取
- 局外成长系统：金币升级数值，金币+材料升阶解锁特性

**单位体系：**
- 基础单位：通关获得，唯一，单一分支进化
- 高级单位：合成/抽卡/黑市获得，可重复获取，支持多分支进化
- 特殊单位：抽卡/黑市获得，可重复获取，支持多分支进化
- 关卡内同一单位只能携带一个，不同分支可同时携带

**攻击类型设计：**
- 弹射类（远程单体稳定输出）
- 近战类（前排高伤爆发）
- 穿透类（一穿多群清）
- 范围类（AoE 高伤）
- 辅助类（增益/控制，无直接伤害）
- 特殊机制类（肉盾/路径改变等策略性玩法）
- 瞬发型（整行爆发伤害，类似一整行秒杀）

**产电系统：**
- 产电玩具为可放置单位，不同场景有不同的产电玩具
- 类型：太阳能类、风力类、热能类
- 不同类型在不同场景有不同机制和产出效率

**合作模式：**
- 双人合作，共享资源池和单位
- 任一防线被突破则双方失败
- 好友模式完全共享，随机匹配采用信任机制（可配置是否需要确认）
- 合作模式提高金币和材料掉落概率以鼓励联机
- 双方各有一个"小手"，独立计算 CD

**小手机制：**
- 关卡内可移动已放置的玩具位置
- 有 CD 冷却，防止频繁操作

**图鉴系统：**
- 局内首次遇到进攻方时解锁图鉴
- 首次获得防守方时解锁图鉴
- 局内弹出图鉴提示，不暂停游戏

**关卡设计：**
- 5 个章节（阳台、客厅、阁楼、玩具店、游乐园），每章 10 关
- 每章第 1 关为简单入门关，学习新场景机制
- 每章第 10 关为 Boss 关（黑暗玩具）
- 难度逐章逐关递进
- 怪物持续出现，波次为同时大量涌出的高峰
- 局内首次遇到新进攻方时弹出图鉴提示（不暂停游戏）
- 无存档机制，退出即丢失局进度

**后续规划（不在本次实现范围）：**
- 黑暗玩具进化/净化系统
- 对战模式（PvP）
- 黑市商人系统
- 局内临时强化机制

## Capabilities

### New Capabilities
- `core-gameplay`: 核心塔防玩法，包括跑道系统、放置机制、波次系统、胜负判定
- `unit-system`: 玩具单位体系，包括基础/高级/特殊单位分类、获取方式、携带规则
- `upgrade-system`: 局外升级升阶系统，包括金币消耗、材料收集、分支进化
- `resource-system`: 局内电力资源系统，包括产电玩具设计、场景差异化
- `combat-system`: 战斗系统，包括 6+1 种攻击类型、伤害计算、技能效果
- `level-design`: 关卡设计系统，包括 5 章节 × 10 关的结构、难度曲线、Boss 战
- `co-op-mode`: 双人合作模式，包括资源共享、匹配机制、信任系统
- `hand-mechanic`: 小手机制，包括移动单位、CD 管理

### Modified Capabilities
（无，这是全新项目）

## Impact

- 技术栈：Unity 6.3 LTS (6000.3.x) + C#
- 渲染方式：2.5D（类似 PvZ 视角，2D 精灵 + Sorting Layer 营造深度）
- 组件架构：MonoBehaviour（非 ECS/DOTS）
- 输入系统：Unity New Input System（支持键盘、鼠标、手柄、触屏）
- 联网框架：Mirror Networking
- 存档系统：JSON 文件（Application.persistentDataPath）
- 场景管理：多场景架构（MainMenu / Game / Loading）
- 对象池：管理弹丸、怪物、电力球等频繁创建销毁的对象
- 版本控制：Git
- 代码规范：Unity 官方 C# 编码规范
- 项目类型：全新项目，需要从零搭建
- 跨平台支持：Windows/Mac/Linux/iOS/Android/Web
- 需要设计数据模型（ScriptableObject 单位/关卡配置）
- 需要美术资源（玩具、怪物、场景）

## Non-goals

- 不实现主角系统（不模仿美食大战老鼠的主角参与防守）
- 不实现对战模式（PvP 作为后续规划）
- 不实现黑市商人系统（后续规划）
- 不实现黑暗玩具净化/进化系统（后续规划）
- 不实现局内临时强化（后续规划）
- 不实现 3D 游戏效果
